### 1. 日志规范问题（严重）

| 问题 | 文件 | 说明 |
|------|------|------|
| ❌ 缺少日志配置 | 所有 Notebook 文件 | 没有导入 `logging` 模块 |
| ❌ 缺少 ipynbname 导入 | 所有 Notebook 文件 | 没有导入 `ipynbname` 模块 |
| ❌ 缺少 logger 配置 | 所有 Notebook 文件 | 没有创建 `logger` 对象 |
| ❌ 缺少配套 .log 文件 | 所有 Notebook 文件 | 没有生成对应的 `.log` 文件 |

**规范要求：**
- 每个脚本/notebook 必须配套输出 log 文件
- log 文件与代码文件同名，扩展名改为 `.log`
- 统一使用 Python 标准库 `logging`

**示例代码：**
```python
import logging
import ipynbname

log_file_name = ipynbname.name() + ".log"
logging.basicConfig(
    level=logging.INFO,
    format="%(asctime)s [%(levelname)s] %(message)s",
    handlers=[
        logging.FileHandler(log_file_name, encoding="utf-8"),
        logging.StreamHandler()
    ]
)
logger = logging.getLogger(__name__)
```

### 2. Jupyter Notebook 结构问题（严重）

| 问题 | 文件 | 说明 |
|------|------|------|
| ❌ 缺少配置区标记 | 所有 Notebook 文件 | 没有明确的配置区标记 |
| ❌ 缺少批量路径获取区标记 | 所有 Notebook 文件 | 没有批量路径获取区标记 |
| ❌ 缺少步骤区标记 | 所有 Notebook 文件 | 没有步骤区标记 |
| ❌ 缺少验证区标记 | `01_fiftyone_current_view_export_640_sub_images_and_labels.ipynb` | 没有验证区标记 |

**规范要求：**
每个 pipeline 的 `.ipynb` 必须按以下顺序组织 cell：

```
# ── 0. 配置区 ──────────────────────────────────────────────
  所有路径、参数、开关集中在这里，方便Master统一修改

# ── 1. 获取批量路径 ─────────────────────────────────────────
  通过代码扫描/筛选，打印出待处理的文件路径列表
  执行后由Master人工确认，手动复制路径到下方配置区再继续

# ── 2. Step N：[动词 + 具体操作] ────────────────────────────
  输入：来自哪里
  处理：做了什么
  输出：存到哪里
  （每个步骤一个独立 cell 块，步骤之间加分隔注释）

# ── 3. 验证 ────────────────────────────────────────────────
  用 display() 抽查输出样本，确认结果符合预期
```

### 3. 代码展示方式问题（中等）

| 问题 | 文件 | 说明 |
|------|------|------|
| ⚠️  使用 print() 展示结构化内容 | `01_fiftyone_current_view_export_640_sub_images_and_labels.ipynb` | 使用 print() 展示数据结构 |

**规范要求：**
- 优先使用 `IPython.display`（`display()`、`display(HTML())`、`display(df)` 等）展示数据
- 不用 `print()` 展示结构化内容

**问题代码示例：**
```python
print(f"export files: {export_dir / 'images'},\n {export_dir / '01_all_labels.json'}")
print(stats)
```

**应改为：**
```python
from IPython.display import display, HTML
display(HTML(f"<p>export files: {export_dir / 'images'},<br> {export_dir / '01_all_labels.json'}</p>"))
display(stats)
```

```python
    raise   # 视情况决定是否继续往上抛
```
